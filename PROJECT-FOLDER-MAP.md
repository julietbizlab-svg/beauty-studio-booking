# 專案資料夾說明

本文件用來說明 `beauty-studio-booking` 商品母版裡，每個主要資料夾與文件的用途。

## 一、主要資料夾

| 名稱 | 用途 | 是否可隨意改名 |
|---|---|---|
| backend | 後端 API，負責連接 Notion、處理預約、取消、業主權限與資料讀寫 | 不可隨意改名 |
| customer-ui | 客人預約畫面，讓客人從 LINE 開啟後可以預約、查詢、取消 | 不可隨意改名 |
| owner-admin | 業主後台，讓業主從 LINE 開啟後可以看預約、改服務、改營業時間 | 不可隨意改名 |
| docs | GitHub Pages 上線用資料夾，網站實際發布會用到這裡的檔案 | 不可隨意改名 |
| scripts | 同步與檢查工具，例如把前端同步到 docs | 不可隨意改名 |
| product-docs | 商品化文件，包含報價、客戶資料蒐集表、交付清單 | 不可隨意改名 |

## 二、主要文件

| 檔案 | 用途 | 是否可隨意改名 |
|---|---|---|
| README.md | 專案主要說明 | 不建議改名 |
| PRODUCT-MASTER.md | 商品母版定位說明 | 不建議改名 |
| CLIENT-SETUP-GUIDE.md | 客戶設定流程說明 | 不建議改名 |
| COPY-FOR-NEW-CLIENT.md | 複製給新客戶時的操作流程 | 不建議改名 |
| PROJECT-FOLDER-MAP.md | 本檔案，說明每個資料夾的用途 | 不建議改名 |

## 三、使用原則

這一整包是商品母版。

平常可以打開文件閱讀，但不要自己任意：

- 改資料夾名稱
- 刪除資料夾
- 搬動檔案位置
- 修改程式碼
- 把客戶機密資料放進母版

## 四、第二個客戶的正確做法

如果有第二個業主要訂這套系統，不是在母版裡直接修改。

正確做法：

1. 複製 `beauty-studio-booking`
2. 建立新的客戶專案資料夾
3. 替換客戶的店名、顏色、服務、營業時間
4. 建立客戶專屬 Notion
5. 建立客戶專屬 LINE LIFF
6. 建立客戶專屬 Cloudflare Worker
7. 測試完成後交付

母版保留乾淨，客戶版另外修改。
