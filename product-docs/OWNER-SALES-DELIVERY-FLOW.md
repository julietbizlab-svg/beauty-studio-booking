# 接案總流程（老闆版）

> **給誰看**：接案老闆、業務窗口——**不是**給客戶或工程師的技術手冊。  
> **一句話**：這份文件告訴你「從客戶問起，到交件收尾」該怎麼想、怎麼問、怎麼報價。

詳細文件可對照：

- [CLIENT-INFO-FORM.md](CLIENT-INFO-FORM.md)（客戶要填什麼）
- [PRICING-PACKAGES.md](PRICING-PACKAGES.md)（三種報價）
- [CLIENT-DELIVERY-SOP.md](CLIENT-DELIVERY-SOP.md)（工程 10 步交付）

---

## 1. 這套系統在賣什麼

**美業一人工作室 LINE 預約管理系統。**

客人用 LINE 選服務、選日期、選時段完成預約；店長用 LINE 看今日預約、改服務、改營業時間、改公告，不用另外裝 App。

---

## 2. 適合賣給誰

- 美甲師
- 美睫師
- 美容師、護膚工作室
- 紋繡師、霧眉師
- 個人工作室、在家接客
- 小型、以「預約制」為主的服務業（一個老闆自己管店）

**理想客戶特徵**：有 LINE 官方帳號、每天預約量不算超大、願意用 LINE 管理客人。

---

## 3. 不適合誰

先說清楚，避免做完才後悔：

| 不適合 | 原因 |
|--------|------|
| 大型連鎖店 | 系統為單店、單業主設計 |
| 多員工排班 | 第一版沒有排班表 |
| 需要線上收款／金流 | 第一版不含金流 |
| 需要複雜報表、營收分析 | 第一版只有預約紀錄 |
| 需要 SaaS 多租戶（一個平台很多店） | 這是「一客戶一系統」模式 |
| 要 App Store 上架 | 這是 LINE 網頁版，不是原生 App |

遇到以上需求 → **加價客製**，或老實說現階段不適合。

---

## 4. 客戶第一次詢問時，我要問什麼

用聊天方式問，不必一次問完：

**關於店**

- [ ] 店名、主要服務是什麼？
- [ ] 一天大概幾位客人？一人工作室還是有幫手？
- [ ] 現在用什麼方式預約？（私訊、電話、IG…）

**關於需求**

- [ ] 最想解決什麼？（漏單、半夜回訊息、忘記排程？）
- [ ] 需要線上收款嗎？
- [ ] 需要幾個人都能登入管理嗎？

**關於帳號**

- [ ] 有 LINE 官方帳號嗎？
- [ ] Notion、Cloudflare、GitHub 想用**自己的帳號**，還是請你代建？

**關於預算與時程**

- [ ] 希望什麼時候上線？
- [ ] 預算大概區間？（可先報三方案讓他選方向）

問完這些，就能判斷該推 Demo、買斷還是維護版。

---

## 5. 怎麼判斷報價

不用背技術，記三種就夠：

### 只是展示 Demo

- **情境**：還沒決定簽約，只想看畫面
- **你提供**：帶他走一遍預約流程（共用示範版）
- **價格**：免費～小額（見 [PRICING-PACKAGES.md](PRICING-PACKAGES.md)）
- **注意**：Demo 不是他的正式系統

### 基礎買斷

- **情境**：確定要、願意自己用業主端改服務與時段
- **你提供**：幫他上線專屬系統 + 驗收 + 操作說明
- **價格**：約 NT$ 18,000～38,000（一次性）
- **之後**：小改可以按次收費，或改簽維護

### 建置加維護

- **情境**：不想碰任何設定，希望有人長期顧
- **你提供**：買斷內容 + 每月檢查 + 每月幫改內容額度 + 優先排查
- **價格**：建置約 NT$ 28,000～48,000 + 月費約 NT$ 2,000～6,000
- **適合**：最省事、你最常賺到長期收入的客戶

**簡單記法**：越不想自己弄，價格越高；越要長期顧，加月費。

## 6. 成交後我要做什麼

老闆端只要記 **3 大步**：

1. **收資料**  
   請客戶填 [CLIENT-INFO-FORM.md](CLIENT-INFO-FORM.md)，確認服務、價格、營業時間、品牌色、取消規則。

2. **交給工程上線**  
   把資料表交給 Cursor／Codex 流程，依 [CLIENT-DELIVERY-SOP.md](CLIENT-DELIVERY-SOP.md) 建 Notion、LINE、後端、網頁。

3. **驗收與交件**  
   自己用 LINE 實機點一遍（見第 9 節），通過後交操作說明、說明維護方式、收尾收款。

你不需要自己改程式，只需要確認「客戶要的」和「做出來的」一致。

---

## 7. 工程端會做什麼

以下由工程（Cursor／Codex）處理，**老闆不用自己做**：

| 工程會做 | 老闆不用做 |
|----------|------------|
| 建 Notion 四個資料表 | 不用自己查 Database ID 怎麼貼 |
| 設定 LINE LIFF | 不用自己改程式碼 |
| 部署 Cloudflare 後端、上傳 Secrets | 不用自己碰 token |
| 填前端網址、上 GitHub Pages | 不用自己 wrangler deploy |
| 檢查安全（token 不進前端） | 不用自己判斷有沒有外洩 |

**原則**：涉及 token、密碼、正式上線、推送到 main 的動作，一律交工程流程與 Codex 審查，老闆只確認商業需求與驗收結果。

---

## 8. 客戶要提供什麼資料

請客戶填 [CLIENT-INFO-FORM.md](CLIENT-INFO-FORM.md)，重點如下：

**基本**

- 店名、業主姓名、聯絡方式
- LINE 官方帳號名稱與連結

**服務**

- 服務名稱、價格、時長
- 哪些要開放線上預約

**營業**

- 哪幾天可預約（週一～週日）
- 每天幾點到幾點

**店面呈現**

- 公告文字
- 品牌色
- 取消規則

**權限**

- 業主 LINE 名稱（userId 由系統協助查）

**帳號歸屬（必問）**

- Notion、LINE、Cloudflare、GitHub 要用**客戶自己的帳號**，還是你代建／代管？

資料越完整，上線越快，來回越少。

## 9. 完成後怎麼驗收

用**客戶的 LINE**實機測，勾完才算交件：

### 客人端

- [ ] 從 LINE 打開預約頁，能登入
- [ ] 看得到服務列表
- [ ] 選日期後看得到時段
- [ ] **可以成功預約**
- [ ] 「我的預約」看得到紀錄
- [ ] **可以取消預約**

### 業主端

- [ ] 業主 LINE 能開啟管理頁（非業主應被擋下）
- [ ] **可以看今日預約**
- [ ] **可以改服務**（新增、下架）
- [ ] **可以改營業時段**
- [ ] **可以改公告與品牌色**

全部通過 → 可參考 [DEMO-ACCEPTANCE-2026-07-14.md](DEMO-ACCEPTANCE-2026-07-14.md) 寫一份驗收紀錄留檔。

## 10. 交付時要跟客戶說什麼

建議當面或訊息講清楚，避免日後誤會：

**帳號與資料**

- 「這套系統用的是**您自己的** LINE、Notion、Cloudflare 設定（或約定好歸誰管）。」
- 「預約資料在您的 Notion，**資料是您的**。」

**安全**

- 「金鑰、密碼只放在後端，**不會寫進客人看得見的網頁**。」
- 「我們不會把 Notion Token 傳給您或貼在聊天裡。」

**系統範圍（第一版）**

- 「第一版專注在 **LINE 預約與管理**，**不含**線上收款、多員工排班、複雜報表。」
- 「若之後要加金流、推播、多店，要**另外報價**。」

**維護**

- 「買斷後小改可以按次；若想長期有人顧，可以簽月維護。」

## 11. 什麼情況要加價

以下不要免費順便做，請另報：

| 需求 | 說明 |
|------|------|
| 多員工／多角色權限 | 第一版只有單一業主 |
| 金流、訂金、線上付款 | 需串接第三方 |
| 營收報表、匯出 Excel | 需額外開發 |
| 客製視覺設計 | 超出基本品牌色 |
| 多店、多分店 | 一店一系統為原則 |
| LINE 自動通知、推播 | 需 Messaging API 等 |
| 與 IG、Google 日曆同步 | 整合專案 |
| 大量舊資料匯入 | 按筆數或工時 |

**口訣**：清單外 = 加價；不確定就先報價再開工。

---

## 12. 老闆自己的提醒

1. **不要低價接太多客製**  
   每多一個「順便加一下」，都是你的時間。先報價，再承諾。

2. **不要把 Demo 帳號當客戶正式帳號**  
   示範版是拿來成交的；簽約後要用**客戶專屬** Notion、LINE、網址。

3. **每個客戶都要獨立帳號與資料**  
   不要多個店共用同一套 Token 或同一個 Notion，以免資料混在一起、安全出事。

4. **有安全疑慮先停下來，交給 Codex 審查**  
   例如：密碼要貼哪、能不能 commit、要不要 deploy——不用自己猜，交工程流程處理。

5. **驗收通過再說「上線完成」**  
   自己用 LINE 點過一輪，比口頭保證有用。

6. **留驗收紀錄**  
   哪天客戶說「以前都可以」，你有紀錄可查。
